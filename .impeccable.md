## Design Context

### Users

GeekTrend serves developers, AI tooling researchers, and technical readers who
want to understand which AI coding agents are appearing in real GitHub Trending
contribution activity. They usually arrive looking for a fast signal first, then
want access to the underlying repository evidence.

### Brand Personality

Analytical, signal-first, and quietly futuristic. The interface should feel like
a research terminal for public developer activity rather than a marketing
landing page.

### Aesthetic Direction

Use a research-terminal / signal-radar visual language: dark ink background,
structured grids, precise metrics, acidic green and amber data highlights, and
subtle scanline or radar details. Avoid generic AI-era blue-purple gradients,
glassmorphism, repetitive rounded card grids, and decorative glow effects that
do not help explain the data.

### Design Principles

- Lead with the strongest live signal: AI agent adoption in the latest snapshot.
- Make provenance visible: users should see snapshot time and raw data context.
- Treat every chart as evidence, not ornament.
- Use motion sparingly to reveal state and hierarchy, with reduced-motion
  support.
- Preserve the static, low-maintenance GitHub Pages architecture.
